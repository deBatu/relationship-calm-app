# Konflikt-Kompass (MVP)

Mobile-first Web-App/PWA fuer zwei Personen, um wiederkehrende Konflikte ruhig und respektvoll zu reflektieren.

## 1) Projekt starten

1. Dateien in einen Webserver-Ordner legen (lokal z. B. mit Live Server oder `npx serve`).
2. In `config.js` die Supabase-Werte eintragen:

```js
window.APP_CONFIG = {
  SUPABASE_URL: "https://YOUR-PROJECT.supabase.co",
  SUPABASE_ANON_KEY: "YOUR_SUPABASE_ANON_KEY",
};
```

## 2) Supabase einrichten (ohne Login)

Im Supabase SQL Editor ausfuehren:

```sql
create table if not exists reflections (
  id bigint generated by default as identity primary key,
  relationship_code text not null check (relationship_code ~ '^[0-9]{6}$'),
  author_id text not null,
  trigger_type text not null,
  feeling text not null,
  need_text text not null,
  created_at timestamptz not null default now()
);

create table if not exists commitments (
  id bigint generated by default as identity primary key,
  relationship_code text not null check (relationship_code ~ '^[0-9]{6}$'),
  author_id text not null,
  commitment_text text not null,
  updated_at timestamptz not null default now(),
  unique (relationship_code, author_id)
);

alter table reflections enable row level security;
alter table commitments enable row level security;

drop policy if exists "anon_select_reflections" on reflections;
drop policy if exists "anon_insert_reflections" on reflections;
drop policy if exists "anon_select_commitments" on commitments;
drop policy if exists "anon_insert_commitments" on commitments;
drop policy if exists "anon_update_commitments" on commitments;

create policy "anon_select_reflections"
on reflections
for select
to anon
using (true);

create policy "anon_insert_reflections"
on reflections
for insert
to anon
with check (true);

create policy "anon_select_commitments"
on commitments
for select
to anon
using (true);

create policy "anon_insert_commitments"
on commitments
for insert
to anon
with check (true);

create policy "anon_update_commitments"
on commitments
for update
to anon
using (true)
with check (true);
```

### Hinweis zur Privatsphaere

Ohne Benutzerkonten kann der 6-stellige Code als gemeinsamer Zugang dienen, ist aber keine starke Sicherheit. Fuer mehr Schutz kannst du spaeter serverseitige Code-Hashing-Logik oder zusaetzliche Secrets einbauen.

## 3) Ablauf in der App

- Session ueber 6-stelligen Code oeffnen
- Trigger-Reflexion speichern
- Partner-Antwort per manuellem Reload laden
- Mini-Commitment getrennt pro Person speichern und beide anzeigen

Synchronisation ist absichtlich asynchron (kein Live-Chat, keine Push-Nachrichten).

## 4) Auf Homescreen legen (PWA)

### Android (Chrome)

1. App-URL in Chrome oeffnen
2. Menue (3 Punkte) -> **Zum Startbildschirm hinzufuegen**
3. Namen bestaetigen -> **Hinzufuegen**

### iPhone/iPad (Safari)

1. App-URL in Safari oeffnen
2. Teilen-Icon -> **Zum Home-Bildschirm**
3. Namen bestaetigen -> **Hinzufuegen**
